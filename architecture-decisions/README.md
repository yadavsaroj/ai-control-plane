# Purpose

AI Control Plane Proposals (ACPs) capture significant architectural proposals that shape the evolution of the AI Control Plane.

They serve a similar purpose to RFCs, PEPs, and Kubernetes Enhancement Proposals by documenting architectural intent before implementation.

---

# Proposal Lifecycle

Every ACP follows the same lifecycle:

```
Draft
    ↓
Review
    ↓
Accepted
    ↓
Implemented
    ↓
Referenced by the AI Control Plane Principles
```

This ensures that architectural principles emerge from implementation experience rather than being defined upfront.

---

# Proposal Structure

Each ACP includes:

* Context
* Problem Statement
* Proposal
* Alternatives Considered
* Trade-offs
* Open Questions
* Consequences
* Related Documents
